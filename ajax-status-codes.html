Understanding AJAX Status Codes

In AJAX, the XMLHttpRequest object provides a status property that returns the HTTP status code of the request. This status code helps determine the result of the request, such as whether it was successful, encountered an error, or was redirected.

Status Codes Overview:

Below are some common HTTP status codes returned by the XMLHttpRequest object:

Successful Status Codes

200OK: The request was successful.
201Created: The request was successful, and a new resource was created.
202Accepted: The request has been accepted, but not yet completed.
204No Content: The request was successful, but there is no data in the response body.
206Partial Content: The server returned part of the data requested.

Redirection Status Codes

300Multiple Choices: The request has multiple possible responses.
301Moved Permanently: The requested page has been moved to a new URL.
302Found: The requested page is found at a different URL.
304Not Modified: The requested page has not been modified.

Client Error Status Codes

400Bad Request: The server could not understand the request due to invalid syntax.
401Unauthorized: The request requires authentication, but credentials were not provided.
403Forbidden: The server understood the request but refuses to authorize it.
404Not Found: The requested page could not be found on the server.

Server Error Status Codes

500Internal Server Error: The server encountered an error while processing the request.
502Bad Gateway: The server acted as a gateway and received an invalid response from an upstream server.
503Service Unavailable: The server is temporarily unavailable.
504Gateway Timeout: The server did not receive a timely response from an upstream server.

Example: Using the Status Code in AJAX

In the following example, we will create an AJAX request to fetch data from the server. Based on the status code returned, we will display different messages.

    
       <script>
          function ShowDoc() {
             // Create an XMLHttpRequest object
             var myhttp = new XMLHttpRequest();
    
             // Create callback function to handle the request
             myhttp.onreadystatechange = function() {
                // Check the status of the response
                if (this.status == 200) {
                   console.log("Found the requested data");
                   document.getElementById("example").innerHTML = this.responseText;
                } else if (this.status == 404) {
                   console.log("Error: The page was not found.");
                   document.getElementById("example").innerHTML = "Error: Page Not Found";
                }
             };
    
             // Open the request with the GET method and the URL
             myhttp.open("GET", "https://jsonplaceholder.typicode.com/todos/3", true);
    
             // Send the request to the server
             myhttp.send();
          }
       </script>

Conclusion:

The status code property in XMLHttpRequest is essential for understanding the result of a request. By using these status codes, we can handle success, error, and redirection scenarios effectively. Depending on the status code, we can perform different actions, such as displaying the data, handling errors, or redirecting the user.
